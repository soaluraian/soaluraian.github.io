255.255.255.248 atau 11111111.11111111.11111111.11111000 atau /29 = subnet mask

192.168.100.24 = net id

Penjelasan:

cara menghitung net id:

2^y - 2 dimana y = jumlah angka 0 pada subnet mask.

255.255.255.248 = 11111111.11111111.11111111.11111000

jumlah angka 0 = 3

2^3 - 2

= 8 - 2

= 6

jumlah host = 6

net id = jumlah host + 2 = 8

192.168.100.0 = net id

192.168.100.1 - 6 = host

192.168.100.7 = ip broadcast

192.168.100.8 = net id

192.168.100.9 - 14 = host

192.168.100.17 - 22 = host

192.168.100.25 - 30 = host

192.168.100.31 = ip broadcast

29. Jelaskan ruang alamat IP untuk jaringan pribadi!​


Jawaban:

Ruang alamat IP untuk jaringan pribadi merujuk pada kumpulan alamat IP yang digunakan secara khusus untuk jaringan lokal atau pribadi di dalam suatu organisasi atau rumah tangga. Jenis alamat IP yang umum digunakan dalam jaringan pribadi adalah IPv4 (Internet Protocol version 4).

Ruang alamat IP untuk jaringan pribadi dibagi menjadi tiga blok utama, yaitu:

1). 10.0.0.0 hingga 10.255.255.255 (10.0.0.0/8)

Blok alamat IP ini memungkinkan untuk mencakup jaringan dengan hingga 16.777.216 (2^24) alamat IP unik. Biasanya digunakan dalam jaringan besar seperti perusahaan atau lembaga pendidikan.

2). 172.16.0.0 hingga 172.31.255.255 (172.16.0.0/12)

Blok alamat IP ini mencakup jaringan dengan hingga 1.048.576 (2^20) alamat IP unik. Biasanya digunakan dalam jaringan menengah hingga besar.

3). 192.168.0.0 hingga 192.168.255.255 (192.168.0.0/16)

Blok alamat IP ini mencakup jaringan dengan hingga 65.536 (2^16) alamat IP unik. Blok ini paling umum digunakan dalam jaringan rumah tangga atau jaringan kecil.

Penjelasan:

Penggunaan ruang alamat IP untuk jaringan pribadi memungkinkan organisasi atau rumah tangga memiliki alamat IP yang unik di dalam jaringan mereka sendiri. Hal ini memungkinkan penggunaan jaringan lokal yang terisolasi dan mempermudah pengelolaan perangkat serta pengaturan keamanan di dalam jaringan tersebut. Namun, alamat IP di dalam ruang alamat IP untuk jaringan pribadi tidak dapat diakses secara langsung dari internet publik. Untuk mengakses internet, jaringan pribadi biasanya menggunakan teknik seperti Network Address Translation (NAT) untuk menerjemahkan alamat IP internal ke alamat IP publik yang terhubung ke internet.

36. Blok alamat IP yang digunakan oleh organisasi untuk menetapkan alamat host internal merupakan pengertian dari​


Blok alamat IP yang digunakan oleh organisasi untuk menetapkan alamat host internal biasanya disebut sebagai "Private IP Address" atau "Private Address Space". Private IP Address adalah rentang alamat IP yang ditetapkan secara eksklusif bagi penggunaan internal di jaringan komputer pribadi.

Ada tiga blok alamat IP yang diperuntukkan untuk penggunaan internal ini yaitu:

1. Blok alamat IP kelas A (10.0.0.0 - 10.255.255.255): Rentang alamat IP ini dapat digunakan oleh organisasi yang memiliki jaringan besar.

2. Blok alamat IP kelas B (172.16.0.0 - 172.31.255.255): Rentang ini diperuntukkan bagi organisasi yang memiliki jaringan berukuran menengah.

3. Blok alamat IP kelas C (192.168.0.0 - 192.168.255.255): Rentang ini sering digunakan oleh rumah tangga dan organisasi kecil yang memiliki jaringan kecil.

Alamat IP dalam blok ini hanya dapat digunakan secara internal di jaringan organisasi. Ketika paket data yang diberi alamat IP private ini akan dikirim melalui internet maka perlu dilakukan proses Network Address Translation (NAT) untuk menggantikan alamat IP private dengan alamat IP publik yang unik agar paket data bisa dikenali oleh router dan jaringan publik.

Dalam pengaturan jaringan blok alamat IP private ini memungkinkan organisasi untuk memiliki kontrol penuh terhadap jaringan internal mereka dan memberikan tambahan keamanan dengan tidak mempublikasikan alamat IP internal secara langsung ke internet.
