1. Cara penghitungan IP subnetting


jumlah subnet, jumlah host per subnet, blok subnet, alamat host dan broadcast yang valid

Jumlah Subnet = 2x, dimana x adalah banyaknya binari 1 pada oktet terakhir subnet mask (2 oktet terakhir untuk kelas B, dan 3 oktet terakhir untuk kelas A). Jadi Jumlah Subnet adalah 22 = 4 subnet

Jumlah Host per Subnet = 2y – 2, dimana y adalah adalah kebalikan dari x yaitu banyaknya binari 0 pada oktet terakhir subnet. Jadi jumlah host per subnet adalah 26 – 2 = 62 host

Blok Subnet = 256 – 192 (nilai oktet terakhir subnet mask) = 64. Subnet berikutnya adalah 64 + 64 = 128, dan 128+64=192. Jadi subnet lengkapnya adalah 0, 64, 128, 192.

Bagaimana dengan alamat host dan broadcast yang valid? Kita langsung buat tabelnya. Sebagai catatan, host pertama adalah 1 angka setelah subnet, dan broadcast adalah 1 angka sebelum subnet berikutnya.

2 . Soal: What is the subnetwork number of a host with an IP address of 172.16.66.0/21?

Jawab: CIDR /21 berarti 255.255.248.0. Blok subnet = 256- 248 = 8, netmasknya adalah kelipatan 8 (0, 8, 16, 24, 32, 40, 48, …, 248) dan karena ini adalah alamat IP kelas B, blok subnet kita “goyang” di oktet ke 3. Tidak perlu kita list semuanya, kita hanya perlu cari kelipatan 8 yang paling dekat dengan 66 (sesuai dengan soal), yaitu 64 dan 72. Jadi susunan alamat IP khusus untuk subnet 172.16.64.0 dan 172.16.72.0 adalah seperti di bawah. Jadi pertanyaan bisa dijawab bahwa 172.16.66.0 itu masuk di subnet 172.16.64.0

14. Proses subnetting untuk IP 192.168.10.0/26 Bantuin dong plis


Dalam proses subnetting untuk IP 192.168.10.0/26, kita akan membagi jaringan tersebut menjadi beberapa subnet dengan ukuran yang lebih kecil. Subnet ini akan memungkinkan pengelompokan perangkat dalam jaringan secara logis, sehingga dapat mempermudah manajemen dan meningkatkan keamanan jaringan.

Pembahasan

Proses subnetting untuk IP 192.168.10.0/26, tahap demi tahap adalah sebagai berikut:

Tentukan jumlah subnet yang diinginkan. Untuk soal ini, kita ingin membuat 4 subnet. Untuk menentukan jumlah subnet yang mungkin, kita perlu menghitung jumlah bit yang digunakan untuk subnet, yaitu 2 bit (karena 2^2 = 4).

Hitung jumlah bit yang digunakan untuk host. Dalam kasus ini, kita menggunakan 6 bit untuk host, sehingga sisa bit untuk subnet adalah 26 - 6 = 20 bit.

Tentukan subnet mask untuk subnet yang diinginkan. Kita menggunakan 2 bit untuk subnet, sehingga subnet mask yang digunakan adalah /26 + 2 = /28.

Hitung jumlah host yang mungkin pada setiap subnet. Kita menggunakan 4 bit untuk host, sehingga jumlah host yang mungkin pada setiap subnet adalah 2^4 - 2 = 14 host (karena dua alamat IP dalam setiap subnet digunakan untuk alamat jaringan dan alamat broadcast).

Tentukan range alamat IP untuk setiap subnet. Pada soal ini, kita membuat 4 subnet dengan range alamat IP sebagai berikut:

Subnet 1: 192.168.10.0/28 (192.168.10.0 - 192.168.10.15)
Subnet 2: 192.168.10.16/28 (192.168.10.16 - 192.168.10.31)
Subnet 3: 192.168.10.32/28 (192.168.10.32 - 192.168.10.47)
Subnet 4: 192.168.10.48/28 (192.168.10.48 - 192.168.10.63)

17. subnetting IP 192.168.10.100/25JIP NID BID RIP​


Subnetting merupakan suatu teknik yang dipakai untuk memecahkan suatu jaringan menjadi beberapa sub jaringan yang lebih kecil. Teknik ini sering kali digunakan untuk mempermudah pengelolaan jaringan.

Pembahasan

Subnetting berperan untuk meningkatkan kinerja serta keamanan suatu jaringan. Akan tetapi, kelemahan dari subnetting yaitu mengambil beberapa perencanaan serta akan memakan waktu yang relatif lama saat menjalani prosesnya. Proses yang digunakan pada metode subnetting dapat dilakukan pada IP Address kelas A, B, dan C. IP Address juga terbagi menjadi beberapa jenis, diantaranya adalah sebagai berikut.

Private IP AddressPublic IP Address

Fungsi utama dari IP Address adalah menangani berbagai koneksi yang dilakukan antar perangkat pengirim maupun penerima melalui suatu jaringan.

Berikut ini adalah beberapa cara untuk mencari JIP, NID, BID, dan RIP.

JIP, dicari dengan cara melihat di table subnetting. Terlihat pada soal menggunakan /25, maka jumlah IP adalah 128. Kemudian, setelah menemukan Jumlah IP Addressnya adalah jumlah IP dikurang dengan 2 (Jumlah IP - 2).NID, dicari dengan cara melakukan operasi AND pada IP Address dan Subnet Mask yang telah diubah menjadi bilangan biner.BID, dicari dengan merubah seluruh bit host (terdapat pada NID, angka biner 1 dengan dirubah menjadi OR).RIP, dicari dengan cara IP NID + 1 sampai dengan IP BID - 1.

18. Jika di kantor kamu hanya terdiri dari 16 komputer sedang mengunakan ip kelas C coba kamu lakukan subnetting jaringan tersebut


Untuk subnetting jaringan dengan 16 komputer menggunakan IP kelas C, kita bisa menggunakan metode subnetting dengan mengubah subnet mask standar (255.255.255.0) menjadi subnet mask yang sesuai dengan jumlah subnet dan jumlah host yang dibutuhkan.

Dalam kasus ini, kita perlu 1 subnet dengan 16 host. Karena 16 tidak merupakan bilangan pangkat 2, maka kita perlu memilih subnet terdekat yang jumlah hostnya cukup untuk menampung 16 host, yaitu 32 host. Oleh karena itu, kita perlu membuat 1 subnet dengan 32 host.

Langkah-langkah subnetting untuk membuat 1 subnet dengan 32 host adalah sebagai berikut:

Tentukan jumlah bit yang dibutuhkan untuk membuat subnet. Karena 2^5 = 32, maka kita perlu menambahkan 5 bit pada subnet mask. Subnet mask menjadi 255.255.255.224 (/27) karena ada 5 bit tambahan.

Hitung jumlah subnet yang dapat dibuat dengan subnet mask tersebut. Jumlah subnet yang dapat dibuat adalah 2^(jumlah bit subnet) = 2^1 = 2 subnet.

Hitung jumlah host yang dapat dibuat pada setiap subnet. Jumlah host yang dapat dibuat adalah 2^(jumlah bit host) - 2, karena 2 alamat IP digunakan untuk alamat jaringan dan alamat broadcast. Jumlah bit host yang tersedia adalah 32-27 = 5 bit, sehingga jumlah host yang dapat dibuat adalah 2^5 - 2 = 30 host.

Tentukan range IP untuk setiap subnet. Range IP untuk subnet pertama adalah 192.168.1.0 - 192.168.1.31 dan range IP untuk subnet kedua adalah 192.168.1.32 - 192.168.1.63.

Dengan subnetting seperti di atas, jaringan dengan 16 komputer dapat diatur menjadi 1 subnet dengan 32 host.

37. Hitunglah Subnetting IP address 192.168.10.10/27​


Untuk melakukan subnetting pada IP address 192.168.10.10/27, pertama-tama kita perlu mengetahui beberapa informasi dasar tentang subnet mask.

Dalam notasi CIDR, /27 menunjukkan jumlah bit yang digunakan untuk mask subnet. Jadi, /27 berarti bahwa 27 bit diperuntukkan untuk mask subnet, sedangkan sisa 5 bit diperuntukkan untuk host.

Sekarang kita dapat menghitung beberapa informasi penting dari subnet ini:

Jumlah subnet yang tersedia: 2^5 = 32Jumlah host yang tersedia per subnet: 2^(32-27) - 2 = 30Network ID: 192.168.10.8 (192.168.10.10 & 255.255.255.224)Broadcast ID: 192.168.10.31 (192.168.10.10 | 0.0.0.31)Rentang IP yang tersedia: 192.168.10.9 - 192.168.10.30

Jadi, dengan subnet mask /27, kita dapat membuat 32 subnet yang masing-masing dapat menampung 30 host. Subnet ini diawali dari alamat IP 192.168.10.8 dan diakhiri dengan alamat IP 192.168.10.31.

39. Jika diberikan IP Address 192.168.170.155/27 lakukan subnetting ! Berapa ? 1. IP BC (IP Awal) 2. IP Net (IP Akhir) 3. IP Host (rentang IP) 4.Subnet Mask 5.Jumlah Subnet 6. Blok Subnet 7. Host per subnet dan beserta caranya​


Jawaban:

IP Address 192.168.170.155/27 adalah alamat IP dengan subnet mask /27, yang menunjukkan bahwa 27 bit dari 32 bit alamat IP tersebut digunakan untuk menentukan subnet, sisanya 5 bit digunakan untuk menentukan host.

1. IP BC (IP Awal): 192.168.170.128

Untuk menentukan IP BC, pertama-tama kita harus menentukan subnet mask dalam bentuk desimal, yaitu 255.255.255.224. Kemudian, kita menentukan subnet dalam bentuk biner, yaitu 11000000.10101000.10101010.10000000. Setelah itu, kita membuat IP BC dengan mengubah hanya satu bit dari bit paling kanan dari subnet menjadi nol, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.128.

2. IP Net (IP Akhir): 192.168.170.159

Untuk menentukan IP Net, kita melakukan hal yang sama dengan menentukan IP BC, tetapi dengan mengubah satu bit dari bit paling kanan dari subnet menjadi satu, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.159.

3. IP Host (rentang IP): 192.168.170.129 - 192.168.170.154

IP Host adalah rentang IP yang dapat digunakan untuk menentukan host pada jaringan, dan dapat ditemukan dengan mengurangi IP BC dengan satu dan menambahkan IP Net dengan satu.

4. Subnet Mask: 255.255.255.224

Subnet Mask adalah alamat IP yang digunakan untuk menentukan bagian dari alamat IP yang digunakan untuk subnet.

5. Jumlah Subnet: 8

Jumlah subnet adalah jumlah subnet yang dapat dibuat dengan menggunakan alamat IP dan subnet mask tertentu. Dalam hal ini, karena kita memiliki 5 bit untuk menentukan host, kita memiliki 2^5 = 32 subnet yang dapat dibuat.

6. Blok Subnet: 32

Blok Subnet adalah jumlah alamat IP yang tersedia pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 = 32 subnet, maka blok subnet adalah 32.

7. Host per subnet: 30

Jumlah host per subnet adalah jumlah host yang dapat ditempatkan pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 - 2 = 30 host per subnet.


40. Hitunglah subnetting IP Address Kelas C dari IP 192.10.10.10 a. Jumlah Subnet​


Untuk melakukan subnetting pada alamat IP Kelas C 192.10.10.10, kita akan membagi alamat IP ini menjadi beberapa subnet. Jumlah subnet yang dapat dibuat tergantung pada seberapa banyak bit host yang akan dialokasikan ke dalam subnet dan berapa banyak bit yang akan digunakan untuk subnetting.

Pada alamat IP Kelas C, biasanya, 24 bit pertama (dari bit 0 hingga bit 23) adalah bagian dari alamat jaringan, dan 8 bit terakhir (dari bit 24 hingga bit 31) adalah bagian dari alamat host. Namun, kita dapat membagi alamat IP ini lebih lanjut dengan meminjam beberapa bit dari alamat host untuk digunakan sebagai subnet.

Misalnya, jika kita ingin menggunakan 4 bit untuk subnetting, maka kita akan meminjam 4 bit dari alamat host, sehingga tersisa 4 bit untuk host. Ini akan menghasilkan 2^4 = 16 subnet yang berbeda.

Jadi, jumlah subnet yang dapat dibuat adalah 16.
