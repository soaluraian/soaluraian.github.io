7. Jelaskan secara singkat mengenai dns dan ip pada istilah ip addres


Pengertian IP address


Alamat IP (Internet Protocol Address atau sering disingkat IP) adalah deretan angka biner antar 32-bit sampai 128-bit yang dipakai sebagai alamat identifikasi untuk tiap komputer host dalam jaringan Internet. Panjang dari angka ini adalah 32-bit (untuk IPv4 atau IP versi 4), dan 128-bit (untuk IPv6 atau IP versi 6) yang menunjukkan alamat dari komputer tersebut pada jaringan Internet berbasis TCP/IP.

Pengertian Domain Name System (DNS)


Domain Name System (DNS) adalah distribute database system yang digunakan untuk pencarian nama komputer (name resolution) di jaringan yang mengunakan TCP/IP (Transmission Control Protocol/Internet Protocol). DNS biasa digunakan pada aplikasi yang terhubung ke Internet seperti web browser atau e-mail, dimana DNS membantu memetakan host name sebuah komputer ke IP address.

13. sebutkan pembagian kelas IP addres


IP address dibagi ke dalam lima kelas, yaitu kelas A, kelas B, kelas C, kelas D dan kelas E. Perbedaan tiap kelas adalah pada ukuran dan jumlahnya. Penentuan kelas ini dilakukan dengan cara berikut : Kelas A : digunakan untuk jaringan WAN, Ip address nya pada bagian pertama antara 0-127,  dan yang merupakan Net ID nya yaitu 1 bagian yang pertama. Subnet mask nya 255.0.0.0  Contoh: 8.254.129.11 Kelas B : biasanya digunakan untuk jaringan MAN, Ip address nya pada bagian pertama antara 128-191, dan yang merupakan network ID nya yaitu 2 bagian pertama. Subnet masknya 255.255.0.0  Contoh: 128.255.129.7 Kelas C : biasanya digunakan untuk jaringan LAN, Ip address nya pada bagian pertama antara 192-223, dan yang merupakan network ID nya yaitu 3 bagian pertama. Subnet masknya 255.255.255.0 Contoh: 192.168.1.10 Kelas D : biasanya digunakan untuk keperluan multicasting. IP address nya pada bagian pertama antara 224-247. Dalam multicasting tidak dikenal network ID dan host ID. Kelas E : biasanya digunakan untuk keperluan umum. IP address nya pada bagian pertama antara 248-255.

24. untuk menghubungkan internet pada komputer melalui jaringan LAN diperlukan penyetingan IP addres. saat melakukan setting IP addres tentunya ada kelebihan dan kekurangannya


Jawaban:

Saat melakukan pengaturan alamat IP pada komputer untuk terhubung ke jaringan LAN, ada beberapa kelebihan dan kekurangan yang perlu diperhatikan. Berikut adalah beberapa di antaranya:

Kelebihan:

1. Identifikasi unik: Setiap perangkat pada jaringan LAN memiliki alamat IP yang unik, sehingga memungkinkan untuk mengidentifikasi dan membedakan satu perangkat dari yang lain.

2. Komunikasi yang efisien: Dengan menggunakan alamat IP, perangkat dapat berkomunikasi secara efisien dalam jaringan LAN, memungkinkan pertukaran data dan sumber daya dengan mudah.

3. Kontrol dan pengelolaan jaringan: Dengan pengaturan IP, administrator jaringan dapat dengan mudah mengendalikan dan mengelola perangkat dalam jaringan, termasuk membatasi akses atau memberikan hak akses tertentu kepada perangkat tertentu.

Kekurangan:

1. Pengaturan yang rumit: Mengatur alamat IP secara manual membutuhkan pemahaman tentang konfigurasi jaringan dan alamat IP. Jika tidak dilakukan dengan benar, dapat menyebabkan masalah dalam koneksi jaringan.

2. Kemungkinan konflik IP: Jika alamat IP yang sama digunakan oleh dua perangkat dalam jaringan, maka akan terjadi konflik IP yang dapat mengganggu koneksi jaringan.

3. Tidak fleksibel: Jika jaringan LAN mengalami perubahan, seperti penambahan perangkat baru atau penggantian perangkat, pengaturan IP perlu diperbarui secara manual untuk mengakomodasi perubahan tersebut.

Penting untuk memahami kelebihan dan kekurangan ini dan melakukannya dengan benar saat melakukan pengaturan IP pada komputer agar koneksi jaringan LAN berjalan dengan baik. Jika Anda tidak yakin atau memiliki kebingungan dalam pengaturan IP, sebaiknya meminta bantuan dari administrator jaringan atau tenaga ahli terkait.

Dynamic Host Configuration Protocol (DHCP) merupakan service yang memungkinkan perangkat dapat mendistribusikan/assign IP Address secara otomatis pada host dalam sebuah jaringan. Cara kerjanya, DHCP Server akan memberikan response terhadap request yang dikirimkan oleh DHCP Client.
