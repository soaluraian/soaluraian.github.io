9. Hitunglah subnetting IP Address Kelas C dari IP 192.10.10.10 a. Jumlah Subnet​


Untuk melakukan subnetting pada alamat IP Kelas C 192.10.10.10, kita akan membagi alamat IP ini menjadi beberapa subnet. Jumlah subnet yang dapat dibuat tergantung pada seberapa banyak bit host yang akan dialokasikan ke dalam subnet dan berapa banyak bit yang akan digunakan untuk subnetting.

Pada alamat IP Kelas C, biasanya, 24 bit pertama (dari bit 0 hingga bit 23) adalah bagian dari alamat jaringan, dan 8 bit terakhir (dari bit 24 hingga bit 31) adalah bagian dari alamat host. Namun, kita dapat membagi alamat IP ini lebih lanjut dengan meminjam beberapa bit dari alamat host untuk digunakan sebagai subnet.

Misalnya, jika kita ingin menggunakan 4 bit untuk subnetting, maka kita akan meminjam 4 bit dari alamat host, sehingga tersisa 4 bit untuk host. Ini akan menghasilkan 2^4 = 16 subnet yang berbeda.

Jadi, jumlah subnet yang dapat dibuat adalah 16.

27. Hitunglah Subnetting IP address 192.168.10.10/27​


Untuk melakukan subnetting pada IP address 192.168.10.10/27, pertama-tama kita perlu mengetahui beberapa informasi dasar tentang subnet mask.

Dalam notasi CIDR, /27 menunjukkan jumlah bit yang digunakan untuk mask subnet. Jadi, /27 berarti bahwa 27 bit diperuntukkan untuk mask subnet, sedangkan sisa 5 bit diperuntukkan untuk host.

Sekarang kita dapat menghitung beberapa informasi penting dari subnet ini:

Jumlah subnet yang tersedia: 2^5 = 32Jumlah host yang tersedia per subnet: 2^(32-27) - 2 = 30Network ID: 192.168.10.8 (192.168.10.10 & 255.255.255.224)Broadcast ID: 192.168.10.31 (192.168.10.10 | 0.0.0.31)Rentang IP yang tersedia: 192.168.10.9 - 192.168.10.30

Jadi, dengan subnet mask /27, kita dapat membuat 32 subnet yang masing-masing dapat menampung 30 host. Subnet ini diawali dari alamat IP 192.168.10.8 dan diakhiri dengan alamat IP 192.168.10.31.

Jika anda butuh subnetting IP address yang lain dan jenis subnetting yang lain, silahkan tanyakan kembali kepada saya.

33. Jika diberikan IP Address 192.168.170.155/27 lakukan subnetting ! Berapa ? 1. IP BC (IP Awal) 2. IP Net (IP Akhir) 3. IP Host (rentang IP) 4.Subnet Mask 5.Jumlah Subnet 6. Blok Subnet 7. Host per subnet dan beserta caranya​


Jawaban:

IP Address 192.168.170.155/27 adalah alamat IP dengan subnet mask /27, yang menunjukkan bahwa 27 bit dari 32 bit alamat IP tersebut digunakan untuk menentukan subnet, sisanya 5 bit digunakan untuk menentukan host.

1. IP BC (IP Awal): 192.168.170.128

Untuk menentukan IP BC, pertama-tama kita harus menentukan subnet mask dalam bentuk desimal, yaitu 255.255.255.224. Kemudian, kita menentukan subnet dalam bentuk biner, yaitu 11000000.10101000.10101010.10000000. Setelah itu, kita membuat IP BC dengan mengubah hanya satu bit dari bit paling kanan dari subnet menjadi nol, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.128.

2. IP Net (IP Akhir): 192.168.170.159

Untuk menentukan IP Net, kita melakukan hal yang sama dengan menentukan IP BC, tetapi dengan mengubah satu bit dari bit paling kanan dari subnet menjadi satu, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.159.

3. IP Host (rentang IP): 192.168.170.129 - 192.168.170.154

IP Host adalah rentang IP yang dapat digunakan untuk menentukan host pada jaringan, dan dapat ditemukan dengan mengurangi IP BC dengan satu dan menambahkan IP Net dengan satu.

4. Subnet Mask: 255.255.255.224

Subnet Mask adalah alamat IP yang digunakan untuk menentukan bagian dari alamat IP yang digunakan untuk subnet.

5. Jumlah Subnet: 8

Jumlah subnet adalah jumlah subnet yang dapat dibuat dengan menggunakan alamat IP dan subnet mask tertentu. Dalam hal ini, karena kita memiliki 5 bit untuk menentukan host, kita memiliki 2^5 = 32 subnet yang dapat dibuat.

6. Blok Subnet: 32

Blok Subnet adalah jumlah alamat IP yang tersedia pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 = 32 subnet, maka blok subnet adalah 32.

7. Host per subnet: 30

Jumlah host per subnet adalah jumlah host yang dapat ditempatkan pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 - 2 = 30 host per subnet.
